The main window of Hash appears as follows:

Item Type Function
File: Text Field Filesystem path to the file whose hash needs to be calculated or verified. The calculated hash for the file is displayed in hexadecimal format, in a field below the buttons.
Verify: Text Field The verification hash, in hexadecimal format, for the specified file. If left blank, the hash will be calculated for the specified file and no verification will be performed.
Select File... Button Brings up a file browser to select the file whose hash needs to be calculated or verified. It is the same as selecting the 'Select File...' menu item from the 'File' menu.
Hash Button Calculates the hash for the specified file. For large files a progress bar is displayed. The progress bar includes a cancel button to stop a hash calculation that is in progress.
Verify Button If a file and a verification hash are both specified, calculates the hash for the specified file and then checks whether it matches the specified verification hash. If the specified verification hash matches the calculated hash, a check mark (✓) is shown next to the calculated hash. Otherwise, an 'X' is shown. For large files a progress bar is displayed. The progress bar includes a cancel button to stop a hash calculation that is in progress.
[Hashes] Menu A menu containing all of the supported hashes. The default is SHA1.
Clear All Button Clears the 'File:' and 'Verify:' fields.

Hash's preferences can be accessed through the 'Preferences...' menu item in the 'Hash' menu:

  • If "Display hash in lowercase" is checked, the calculated hash for the specified file is shown using lower case letters. The default is to use upper case letters.
  • If "Display file size after hash" is checked, the specified file's size (in bytes) is shown in parentheses after the calculated hash. The default is to omit the file size.
